# AI 코드 검증 체크리스트

> AI가 생성한 코드를 적용하기 전에 이 체크리스트를 확인하세요.

---

## 1. 버전 호환성 검증

### 필수 확인
- [ ] **import 경로** 확인
  - 라이브러리 import 경로가 현재 버전과 일치하는가?
  - 예: `@supabase/supabase-js` (O) vs `@supabase/supabase-js/dist/main` (X)

- [ ] **API 메서드** 확인
  - 사용된 메서드가 현재 버전에서 지원되는가?
  - 공식 문서에서 메서드 존재 여부 확인

- [ ] **메서드 시그니처** 확인
  - 파라미터 순서와 타입이 올바른가?
  - 반환값 구조가 올바른가?

- [ ] **deprecated 경고** 확인
  - 콘솔에 deprecated 경고가 없는가?
  - 공식 문서에서 deprecated 표시가 없는가?

### 버전 확인 방법
```bash
# package.json 확인
cat package.json | grep "라이브러리명"

# 설치된 버전 확인
npm list 라이브러리명
```

---

## 2. 실행 테스트

### 기본 테스트
- [ ] **컴파일/빌드 성공**
  - 문법 에러 없이 빌드되는가?
  - TypeScript 타입 에러가 없는가?

- [ ] **런타임 에러 없음**
  - 코드가 에러 없이 실행되는가?
  - 콘솔에 에러 메시지가 없는가?

- [ ] **예상 동작 확인**
  - 의도한 대로 동작하는가?
  - UI가 정상적으로 렌더링되는가?

### 엣지 케이스 테스트
- [ ] **빈 값 처리**
  - null, undefined, 빈 배열/객체 처리가 되는가?

- [ ] **에러 상황 처리**
  - 네트워크 에러 시 적절히 처리되는가?
  - 사용자에게 에러 메시지가 표시되는가?

---

## 3. 코드 품질 검증

### 코딩 컨벤션
- [ ] **네이밍 규칙** 준수
  - 변수명, 함수명이 프로젝트 규칙을 따르는가?
  - 명확하고 이해하기 쉬운 이름인가?

- [ ] **코드 스타일** 준수
  - ESLint/Prettier 경고가 없는가?
  - 들여쓰기, 공백이 일관되는가?

### 불필요한 코드
- [ ] **미사용 변수/import 없음**
  - 사용하지 않는 변수가 선언되어 있지 않은가?
  - 불필요한 import가 없는가?

- [ ] **중복 코드 없음**
  - 기존 코드와 중복되는 로직이 없는가?

---

## 4. 보안 검증

### 입력 검증
- [ ] **사용자 입력 검증**
  - 사용자 입력이 적절히 검증되는가?
  - 길이 제한, 형식 검사가 있는가?

- [ ] **XSS 방지**
  - innerHTML 대신 textContent 사용하는가?
  - dangerouslySetInnerHTML 사용 시 sanitize 하는가?

### 인증/권한
- [ ] **민감 정보 노출 없음**
  - API 키, 비밀번호가 하드코딩되지 않았는가?
  - 환경 변수로 관리되는가?

- [ ] **권한 검사**
  - 적절한 권한 검사가 수행되는가?
  - RLS에만 의존하지 않고 UI에서도 처리하는가?

---

## 5. 공식 문서 대조

### 대조 항목
- [ ] **API 사용법** 확인
  - 공식 문서의 예제와 일치하는가?
  - 최신 문서를 참조했는가?

- [ ] **Best Practices** 확인
  - 공식 권장 패턴을 따르는가?
  - Anti-pattern을 피했는가?

### 참조할 문서
| 라이브러리 | 공식 문서 URL |
|-----------|---------------|

| Supabase | https://supabase.com/docs |

2. **문제 발견 시 기록하세요**
   - AI 사용 로그에 발견된 문제와 해결 방법 기록
