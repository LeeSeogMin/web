# 유저 스토리 템플릿

## 에픽 (Epic)
> 큰 기능 단위를 정의합니다.

**에픽 ID**: EPIC-001
**에픽 제목**: 게시판 기능
**설명**: 사용자가 게시글을 작성, 조회, 수정, 삭제할 수 있는 기능

---

## 유저 스토리 (User Story)

### 스토리 형식
```
As a <역할>,
I want <기능>,
so that <가치/이유>.
```

---

### 스토리 예시

#### US-001: 게시글 목록 조회
**역할**: 방문자
**기능**: 게시글 목록을 볼 수 있다
**가치**: 어떤 글들이 있는지 파악하기 위해

**인수 조건**:
- [ ] Given: 게시판 페이지에 접속했을 때
- [ ] When: 페이지가 로드되면
- [ ] Then: 게시글 제목, 작성자, 작성일이 목록으로 표시된다
- [ ] Then: 최신 글이 상단에 표시된다

**우선순위**: Must Have

---

#### US-002: 게시글 작성
**역할**: 로그인한 회원
**기능**: 새 게시글을 작성할 수 있다
**가치**: 내 의견이나 정보를 공유하기 위해

**인수 조건**:
- [ ] Given: 로그인한 상태에서
- [ ] When: 제목과 내용을 입력하고 등록 버튼을 클릭하면
- [ ] Then: 게시글이 저장되고 목록에 표시된다
- [ ] Then: 작성자 정보가 자동으로 기록된다

**우선순위**: Must Have

---

#### US-003: 게시글 수정
**역할**: 게시글 작성자
**기능**: 내가 작성한 게시글을 수정할 수 있다
**가치**: 오류를 수정하거나 내용을 보완하기 위해

**인수 조건**:
- [ ] Given: 내가 작성한 게시글 상세 페이지에서
- [ ] When: 수정 버튼을 클릭하면
- [ ] Then: 편집 가능한 폼이 표시된다
- [ ] When: 내용을 수정하고 저장하면
- [ ] Then: 변경사항이 반영된다

**우선순위**: Should Have

---

#### US-004: 게시글 삭제
**역할**: 게시글 작성자 또는 관리자
**기능**: 게시글을 삭제할 수 있다
**가치**: 부적절하거나 불필요한 글을 제거하기 위해

**인수 조건**:
- [ ] Given: 삭제 권한이 있는 게시글에서
- [ ] When: 삭제 버튼을 클릭하면
- [ ] Then: 삭제 확인 대화상자가 표시된다
- [ ] When: 확인하면
- [ ] Then: 게시글이 삭제되고 목록으로 이동한다

**우선순위**: Should Have

---

## MoSCoW 우선순위 분류

| 분류 | 스토리 |
|------|--------|
| **Must Have** | US-001 목록 조회, US-002 게시글 작성 |
| **Should Have** | US-003 수정, US-004 삭제 |
| **Could Have** | 댓글, 좋아요, 검색 |
| **Won't Have** | 파일 첨부, 알림 (다음 버전) |
