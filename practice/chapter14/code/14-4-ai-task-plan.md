## 2. AI 위임 기준

### AI에게 위임 가능한 작업

| 작업 유형 | 예시 | 검증 수준 |
|-----------|------|-----------|
| **보일러플레이트** | 컴포넌트 구조, import 문 | 낮음 |
| **반복 코드** | CRUD 함수, 폼 핸들러 | 중간 |
| **스타일링** | CSS, Tailwind 클래스 | 낮음 |
| **유틸리티** | 날짜 포맷, 에러 메시지 | 중간 |
| **문서화** | 주석, README | 낮음 |

### 인간이 반드시 수행할 작업

| 작업 유형 | 이유 |
|-----------|------|
| **보안 정책** | RLS, 인증 흐름은 검증 필수 |
| **API 키 관리** | 환경 변수 직접 설정 |
| **아키텍처 결정** | 폴더 구조, 상태 관리 방식 |
| **최종 코드 리뷰** | AI 출력물 검증 |
| **배포 작업** | 프로덕션 환경 설정 |

## 3. 검증 포인트

### 코드 생성 후 체크리스트

```markdown
□ 1. 버전 호환성
   - 사용된 API가 현재 라이브러리 버전과 맞는가?
   - deprecated 경고가 없는가?

□ 2. 보안 검토
   - 하드코딩된 시크릿이 없는가?
   - XSS 취약점이 없는가?
   - SQL 인젝션 가능성이 없는가?

□ 3. 스타일 일관성
   - 기존 코드 컨벤션을 따르는가?
   - import 경로가 올바른가?

□ 4. 실제 동작 확인
   - 브라우저에서 실행해 보았는가?
   - 콘솔 에러가 없는가?
   - 예상대로 동작하는가?

□ 5. 엣지 케이스
   - 빈 데이터 처리가 되어 있는가?
   - 에러 상황 처리가 되어 있는가?
```

### 단계별 리뷰 포인트

| 단계 | 리뷰 항목 |
|------|-----------|
| 환경 설정 후 | 개발 서버 정상 실행 확인 |
| 인증 구현 후 | 로그인/로그아웃 플로우 테스트 |
| CRUD 구현 후 | 모든 작업 정상 동작 확인 |
| 보안 적용 후 | 권한 없는 접근 차단 확인 |
| 배포 후 | 프로덕션 환경 기능 테스트 |
