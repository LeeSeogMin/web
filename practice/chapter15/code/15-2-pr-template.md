# Pull Request 템플릿

## PR 제목 형식
```
[타입] 간단한 설명

예시:
[feat] 게시글 작성 기능 구현
[fix] 로그인 후 리다이렉트 오류 수정
[refactor] PostList 컴포넌트 분리
[style] 버튼 호버 효과 추가
[docs] README 설치 가이드 추가
```

## PR 타입 목록

| 타입 | 설명 |
|------|------|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `refactor` | 코드 리팩토링 (기능 변경 없음) |
| `style` | 스타일, CSS 변경 |
| `docs` | 문서 수정 |
| `chore` | 빌드 설정, 패키지 등 기타 변경 |
| `test` | 테스트 코드 추가/수정 |

---

## 좋은 PR의 조건

### ✅ DO
- 작은 단위로 PR 생성 (200줄 이내 권장)
- 명확한 제목과 설명 작성
- 스크린샷/GIF로 변경사항 시각화
- 테스트 방법 안내
- 관련 이슈 연결

### ❌ DON'T
- 여러 기능을 하나의 PR에 포함
- 설명 없이 PR 생성
- 리뷰 요청 전 자체 검토 생략
- 불필요한 파일 변경 포함
