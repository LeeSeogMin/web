ARIA 속성 예제 모음

이 페이지는 자주 사용되는 ARIA 속성의 올바른 사용법을 보여줍니다.

1. aria-label

보이는 텍스트 없이 접근 가능한 이름을 제공할 때 사용합니다.

아이콘 버튼 예제

❌ 잘못된 예 (접근 가능한 이름 없음):

<button>🔍</button>

✅ 올바른 예:

<button aria-label="검색">🔍</button>

스크린 리더는 "검색, 버튼"이라고 읽습니다.

2. aria-labelledby

다른 요소의 텍스트를 라벨로 참조할 때 사용합니다.

폼 그룹 예제

결제 정보

<div role="group" aria-labelledby="group-title">
    <h4 id="group-title">결제 정보</h4>
    ...
</div>

3. aria-describedby

추가 설명을 연결할 때 사용합니다.

8자 이상, 특수문자 포함

비밀번호가 너무 짧습니다.

<input
    type="password"
    id="pw"
    aria-describedby="pw-help pw-error"
>
<p id="pw-help">8자 이상, 특수문자 포함</p>
<p id="pw-error">비밀번호가 너무 짧습니다.</p>

4. aria-hidden

장식용 요소를 스크린 리더에서 숨깁니다.

📧 이메일: contact@example.com 

<span aria-hidden="true">📧</span>
이메일: contact@example.com

아이콘 이모지는 스크린 리더에서 무시됩니다.

5. aria-live

동적으로 변경되는 콘텐츠를 스크린 리더에 알립니다.

저장이 완료되었습니다. 
<div role="status" aria-live="polite">
    저장이 완료되었습니다.
</div>

polite: 현재 읽고 있는 내용 후에 알림

assertive: 즉시 알림 (긴급한 경우만)

6. 탭 인터페이스 (role="tablist")

JavaScript는 웹 브라우저에서 실행되는 스크립트 언어입니다.
Python은 간결한 문법으로 인기 있는 범용 프로그래밍 언어입니다.

7. 진행 상태 (role="progressbar")

70% 완료

<div
    role="progressbar"
    aria-valuenow="70"
    aria-valuemin="0"
    aria-valuemax="100"
    aria-label="파일 업로드 진행률"
>...</div>

ARIA 사용 5대 원칙

  1. 네이티브 HTML 우선: ARIA보다 <button>, <label> 등 사용
  2. 역할 변경 금지: <h1 role="button"> 같은 용법 피하기
  3. 키보드 지원 필수: ARIA는 시맨틱만 변경, 기능은 JS로 구현
  4. 포커스 가능 요소만: 숨긴 요소에 포커스 가지 않도록
  5. 접근 가능한 이름 필수: 모든 인터랙티브 요소에 이름 제공
