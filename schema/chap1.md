# 제1장: 웹이 동작하는 방식 — 집필 계획서

## 장 개요
- **분량**: 600-700줄 (약 60,000-70,000자), A4 35-40쪽
- **구성**: 이론 설명 70%, 코드 예제 30%
- **대상**: 학부생 (선수 지식: 기초 컴퓨터 활용)
- **난이도**: ★☆☆ 기초

## 학습 목표 (3-5개)
이 장을 마치면 학생들은:
1. 클라이언트-서버 아키텍처의 개념을 **이해하고 설명할 수 있다**
2. HTTP 요청/응답의 흐름을 **단계별로 설명할 수 있다**
3. 브라우저 렌더링 과정을 **순서대로 나열할 수 있다**
4. DOM의 역할과 JavaScript 실행 환경을 **이해할 수 있다**
5. DevTools를 활용하여 **네트워크 요청과 DOM 구조를 분석할 수 있다**

## 핵심 용어 (Key Terms)
- **클라이언트(Client)**: 서비스를 요청하는 측 (웹 브라우저)
- **서버(Server)**: 서비스를 제공하는 측 (웹 서버)
- **HTTP**: 웹에서 데이터를 주고받기 위한 프로토콜
- **DOM(Document Object Model)**: HTML을 트리 구조로 표현한 것
- **렌더링(Rendering)**: 브라우저가 HTML/CSS/JS를 화면에 그리는 과정

## 1.1 클라이언트-서버 아키텍처 (6-8쪽)

### 집필 방향
웹의 기본 구조를 학생들에게 친숙한 비유(식당 주문)를 통해 설명하고, 실제 웹 통신이 어떻게 이루어지는지 이해시킨다.

### 학습 순서 설계
1. **왜 배우는가**: "웹사이트를 열 때 무슨 일이 일어날까?" - 호기심 유발
2. **무엇인가**: 클라이언트/서버의 역할과 책임 분리
3. **어떻게 사용하는가**: 실제 웹 요청 예시와 흐름 추적

### 비유/예시 계획
- 클라이언트-서버 → "식당에서 손님(클라이언트)이 주문하고, 주방(서버)이 음식을 만들어 제공"
- URL → "주소"
- 포트 번호 → "아파트 동/호수"

### 내용 구성

**1.1.1 클라이언트와 서버의 역할** (3쪽)
- 핵심 개념: 역할 분리, 요청-응답 모델
- 비유: 식당 손님과 주방
- 학생 질문 예상: "서버는 컴퓨터인가요? 프로그램인가요?"

**1.1.2 URL과 도메인의 이해** (2쪽)
- 핵심 개념: URL 구조 (프로토콜://도메인:포트/경로)
- 실습: 브라우저 주소창에서 URL 분석
- 학생 질문 예상: "www는 왜 붙이나요?"

**1.1.3 포트와 프로토콜** (2쪽)
- 핵심 개념: 포트 번호의 역할, HTTP vs HTTPS
- 비유: 아파트 동/호수
- 이전 개념과 연결: URL의 포트 부분

### 코드 예제 (30%)
- 예제 1.1: URL 분해 시각화
- 사용 도구: 브라우저 DevTools
- 예상 출력: URL 각 부분 설명

### 참고문헌
- MDN Web Docs. "How the Web works"
- 생활코딩. "클라이언트와 서버"

## 1.2 HTTP 요청과 응답의 흐름 (8-10쪽)

### 집필 방향
HTTP가 실제로 어떤 형식으로 데이터를 주고받는지, 요청/응답의 구조를 상세히 설명한다.

### 학습 순서 설계
1. **왜 배우는가**: "브라우저가 서버와 대화하는 방법을 알아야 에러를 해결할 수 있다"
2. **무엇인가**: HTTP 메시지 구조 (헤더, 바디)
3. **어떻게 사용하는가**: DevTools에서 실제 요청/응답 확인

### 비유/예시 계획
- HTTP 요청 → "편지 봉투(헤더)와 편지 내용(바디)"
- 상태 코드 → "택배 배송 상태 (배송 완료, 주소 오류, 품절)"

### 내용 구성

**1.2.1 HTTP 요청의 구조** (3쪽)
- 핵심 개념: 메서드(GET, POST), 헤더, 바디
- 표: HTTP 메서드 비교 (GET vs POST)
- 학생 질문 예상: "GET과 POST의 차이는?"

**1.2.2 HTTP 응답의 구조** (3쪽)
- 핵심 개념: 상태 코드, 응답 헤더, 응답 바디
- 표: 주요 상태 코드 (200, 404, 500 등)
- 학생 질문 예상: "404 에러는 왜 생기나요?"

**1.2.3 요청-응답 사이클** (3쪽)
- 핵심 개념: DNS 조회 → TCP 연결 → HTTP 통신
- 다이어그램: 요청-응답 흐름도
- 이전 개념과 연결: 1.1의 클라이언트-서버

### 코드 예제 (30%)
- 예제 1.2: DevTools Network 탭에서 요청 분석
- 사용 도구: Chrome DevTools
- 예상 출력: 요청/응답 헤더 스크린샷 설명

### 참고문헌
- MDN Web Docs. "HTTP 개요"
- HTTP/1.1 RFC 2616

## 1.3 브라우저 렌더링 파이프라인 (6-8쪽)

### 집필 방향
브라우저가 HTML, CSS, JavaScript를 어떻게 화면에 그리는지 단계별로 설명한다.

### 학습 순서 설계
1. **왜 배우는가**: "왜 어떤 사이트는 느리고 어떤 사이트는 빠를까?"
2. **무엇인가**: 파싱, 렌더 트리, 레이아웃, 페인트
3. **어떻게 사용하는가**: 성능 최적화 힌트

### 비유/예시 계획
- 렌더링 파이프라인 → "건축 과정: 설계도(HTML) → 인테리어(CSS) → 시공(Paint)"
- DOM 트리 → "가계도처럼 부모-자식 관계"

### 내용 구성

**1.3.1 HTML 파싱과 DOM 트리** (3쪽)
- 핵심 개념: HTML → DOM 변환 과정
- 다이어그램: DOM 트리 예시
- 학생 질문 예상: "DOM과 HTML은 같은 건가요?"

**1.3.2 CSS 파싱과 CSSOM** (2쪽)
- 핵심 개념: CSS → CSSOM 변환
- 렌더 트리 = DOM + CSSOM
- 이전 개념과 연결: DOM 트리와 결합

**1.3.3 레이아웃과 페인트** (2쪽)
- 핵심 개념: 레이아웃 계산, 픽셀 페인트
- 성능 최적화 팁: 리플로우 최소화
- 학생 질문 예상: "왜 스크롤이 끊기나요?"

### 코드 예제 (30%)
- 예제 1.3: 간단한 HTML의 DOM 트리 시각화
- 사용 도구: DevTools Elements 탭
- 예상 출력: DOM 트리 스크린샷

### 참고문헌

- MDN Web Docs. "Populating the page"

## 1.4 DOM과 JavaScript 런타임 (6-8쪽)

### 집필 방향
JavaScript가 DOM을 어떻게 조작하는지, 브라우저의 JavaScript 실행 환경을 설명한다.

### 학습 순서 설계
1. **왜 배우는가**: "정적인 웹페이지를 동적으로 만드는 핵심"
2. **무엇인가**: DOM API, 이벤트 루프
3. **어떻게 사용하는가**: 기본 DOM 조작 예제

### 비유/예시 계획
- DOM → "리모컨: HTML 요소를 조종하는 도구"
- 이벤트 루프 → "은행 창구: 한 번에 한 고객씩 처리"

### 내용 구성

**1.4.1 DOM이란 무엇인가** (2쪽)
- 핵심 개념: 문서를 객체로 표현
- 비유: 리모컨으로 TV 조작
- 학생 질문 예상: "DOM은 언제 만들어지나요?"

**1.4.2 JavaScript와 DOM 조작** (3쪽)
- 핵심 개념: querySelector, 이벤트 리스너
- 코드 예제: 버튼 클릭 시 텍스트 변경
- 이전 개념과 연결: 1.3의 DOM 트리

**1.4.3 JavaScript 런타임 환경** (2쪽)
- 핵심 개념: 콜 스택, 이벤트 루프, 태스크 큐
- 비유: 은행 창구 시스템
- 학생 질문 예상: "JavaScript는 왜 비동기인가요?"

### 코드 예제 (30%)
- 예제 1.4: DOM 요소 선택 및 수정
- 사용 언어: JavaScript
- 예상 출력: 버튼 클릭 전/후 화면

## 1.5 DevTools 실습: 네트워크/요소/콘솔 탭 활용 (8-10쪽)

### 실습 목표
- DevTools를 열고 기본 인터페이스를 탐색할 수 있다
- Network 탭에서 HTTP 요청/응답을 분석할 수 있다
- Elements 탭에서 DOM 구조를 확인하고 수정할 수 있다
- Console 탭에서 JavaScript를 실행할 수 있다

### 실습 구성
1. **환경 준비**: Chrome 브라우저, DevTools 열기 (F12)
2. **Network 탭 실습**: 요청 분석, 헤더 확인
3. **Elements 탭 실습**: DOM 탐색, 스타일 수정
4. **Console 탭 실습**: JavaScript 명령 실행

### 필요 도구
- Chrome 브라우저 (최신 버전)
- 인터넷 연결

### 예상 결과물
- 실습 스크린샷
- 분석 보고서 작성
