# 제12장 집필계획서: 에러 처리와 UX

## 1. 장 개요

### 1.1 위치 및 역할
- **Part 5**: 배포와 운영의 첫 번째 장
- **선수 지식**: 10장 CRUD, 11장 RLS
- **후속 장**: 13장 Vercel 배포

### 1.2 핵심 목표
학부생이 웹 애플리케이션에서 발생하는 다양한 에러 유형을 이해하고, try-catch와 Error Boundary로 에러를 처리하며, 로딩 인디케이터와 스켈레톤 UI로 사용자 경험을 개선할 수 있다.

### 1.3 예상 분량
- 목표: 600-700줄 (핵심 개념 장)
- 이론:실습 = 50:50
- A4 약 35쪽

## 3. 학습 목표

1. 웹 애플리케이션에서 발생하는 에러 유형을 분류할 수 있다
2. try-catch로 비동기 에러를 처리할 수 있다
3. Error Boundary로 컴포넌트 에러를 격리할 수 있다
4. 사용자 친화적인 에러 메시지를 설계할 수 있다
5. 로딩 인디케이터와 스켈레톤 UI를 구현할 수 있다

---

## 4. 핵심 용어

| 용어 | 정의 |
|------|------|
| Error Boundary | 하위 컴포넌트 에러를 잡아 대체 UI를 보여주는 React 컴포넌트 |
| try-catch | 예외를 잡아 처리하는 JavaScript 문법 |
| 스켈레톤 UI | 콘텐츠 로딩 중 레이아웃 형태를 미리 보여주는 UI |
| Optimistic UI | 서버 응답 전에 성공을 가정하고 UI를 먼저 업데이트 |
| 토스트 | 화면 구석에 잠시 나타났다 사라지는 알림 메시지 |
