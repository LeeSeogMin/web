# 15장 집필계획서: 프로젝트 구현과 검증

## 1. 장 개요

### 제목
프로젝트 구현과 검증

### 핵심 주제
14장에서 설계한 프로젝트를 실제로 구현하고 검증하는 방법. 스프린트 실행, 코드 리뷰, 테스트, 문서화까지 프로젝트 완성의 전 과정을 다룬다.

### 학습목표
1. 기능별 구현 순서를 결정하고 점진적으로 통합할 수 있다
2. PR을 작성하고 코드 리뷰를 수행할 수 있다
3. 기능 테스트와 RLS 검증을 수행할 수 있다
4. README와 회고 보고서를 작성할 수 있다

### 예상 분량
- 목표: 550-650줄
- 유형: 실습 중심 장 (이론:실습 = 40:60)

---

## 2. 절 구성

### 15.1 스프린트 실행 (약 140줄)
- 15.1.1 기능별 구현 순서
  - 의존성 기반 우선순위
  - 인증 → CRUD → 보안 순서
  - 점진적 빌드 검증
- 15.1.2 점진적 통합
  - 작은 단위로 커밋
  - 기능 완성 후 통합
  - 통합 시 충돌 해결

### 15.2 코드 리뷰 (약 140줄)
- 15.2.1 PR 작성법
  - PR 제목과 설명 작성
  - 변경사항 요약
  - 스크린샷/GIF 첨부
- 15.2.2 리뷰 체크리스트
  - 기능 동작 확인
  - 코드 스타일 검토
  - 보안 취약점 점검
  - AI 생성 코드 검증

### 15.3 최종 검증 (약 150줄)
- 15.3.1 기능 테스트
  - 사용자 시나리오 기반 테스트
  - CRUD 동작 확인
  - 엣지 케이스 테스트
- 15.3.2 RLS 검증
  - 권한 없는 접근 시도
  - 다른 사용자 데이터 접근 차단 확인
  - 관리자 권한 테스트
- 15.3.3 에러 시나리오 테스트
  - 네트워크 오류 처리
  - 잘못된 입력 처리
  - 세션 만료 처리

### 15.4 문서화 (약 140줄)
- 15.4.1 README 작성
  - 프로젝트 소개
  - 설치 및 실행 방법
  - 환경 변수 설정
  - 기술 스택
- 15.4.2 AI 사용 로그 정리
  - 사용한 프롬프트 기록
  - AI 생성 코드 표시
  - 수정/검증 내역
- 15.4.3 회고 보고서
  - 잘된 점 / 개선할 점
  - 학습한 내용
  - 다음 프로젝트 적용 계획

### 장 요약 및 연습문제 (약 70줄)
- 핵심 내용 bullet point
- 연습문제 8문제 (기초 3, 중급 3, 심화 2)
