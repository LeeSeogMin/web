# 14장 집필계획서: 프로젝트 설계

## 1. 장 개요

### 제목
프로젝트 설계

### 핵심 주제
프로젝트 시작 전 체계적인 설계 방법론 학습. 요구사항 정의, 데이터 모델링, 권한 설계, AI 협업 계획까지 프로젝트 성공의 기반을 다진다.

### 학습목표
1. 유저 스토리와 기능 명세서를 작성할 수 있다
2. ERD를 활용하여 데이터 모델을 설계할 수 있다
3. 역할과 권한 매트릭스를 정의할 수 있다
4. AI 협업을 위한 작업 분해와 검증 계획을 수립할 수 있다

### 예상 분량
- 목표: 600-700줄
- 유형: 핵심 개념 장 (이론:실습 = 70:30)

---

## 2. 절 구성

### 14.1 요구사항 정의 (약 150줄)
- 14.1.1 유저 스토리 작성법
  - 유저 스토리 형식: "~로서, ~을 원한다, 왜냐하면 ~이기 때문이다"
  - 에픽과 스토리의 계층 구조
  - 인수 조건(Acceptance Criteria) 작성
- 14.1.2 기능 명세서 템플릿
  - 기능 ID, 이름, 설명, 입력/출력
  - 선행 조건, 예외 케이스
  - 우선순위 설정 (MoSCoW 방법)

### 14.2 데이터 모델 설계 (약 150줄)
- 14.2.1 ERD 작성
  - 엔티티와 속성 식별
  - 관계 표현 (1:1, 1:N, N:M)
  - ERD 다이어그램 도구 소개
- 14.2.2 테이블 스키마 정의
  - 컬럼, 자료형, 제약조건
  - 게시판 예시 스키마 설계
  - Supabase에서의 구현

### 14.3 권한 규칙 설계 (약 130줄)
- 14.3.1 역할 정의
  - 익명 사용자, 일반 회원, 관리자
  - 역할별 접근 범위
- 14.3.2 권한 매트릭스 작성
  - CRUD 별 권한 표
  - RLS 정책으로 변환
  - 권한 충돌 방지

### 14.4 AI 협업 계획 (약 150줄)
- 14.4.1 작업 분해
  - WBS(Work Breakdown Structure) 기초
  - 기능 단위로 분해
  - 의존성 파악
- 14.4.2 AI에게 위임할 작업 선정
  - AI 적합 작업: 보일러플레이트, 반복 코드, 스타일링
  - 인간 필수 작업: 비즈니스 로직, 보안, 아키텍처 결정
  - 협업 비율 가이드
- 14.4.3 검증 포인트 설정
  - 단계별 체크포인트
  - 코드 리뷰 기준
  - AI 출력 검증 체크리스트

### 장 요약 및 연습문제 (약 70줄)
- 핵심 내용 bullet point
- 연습문제 8문제 (기초 3, 중급 3, 심화 2)
