## 3. 학습 목표

1. 서버리스 배포의 개념을 설명할 수 있다
2. GitHub 저장소를 Vercel에 연동하여 배포할 수 있다
3. 환경 변수를 안전하게 설정할 수 있다
4. 배포 로그를 확인하고 문제를 진단할 수 있다
5. 필요시 이전 버전으로 롤백할 수 있다

---

## 4. 핵심 용어

| 용어 | 정의 |
|------|------|
| 서버리스 | 서버 관리 없이 코드만 배포하는 방식 |
| CI/CD | 지속적 통합/배포, 코드 변경 시 자동 배포 |
| 환경 변수 | 배포 환경별로 다른 설정값 |
| 롤백 | 이전 배포 버전으로 되돌리기 |
| 프리뷰 배포 | PR마다 생성되는 테스트 배포 |
