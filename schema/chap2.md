# 제2장: AI 코딩 도구 활용법 — 집필 계획서

## 장 개요
- **분량**: 700-800줄 (약 70,000-80,000자), A4 40-45쪽
- **구성**: 이론 설명 60%, 실습/예제 40%
- **대상**: 학부생 (선수 지식: 1장 웹 기초, 기본 프로그래밍)
- **난이도**: ★★☆ 중급

## 학습 목표 (3-5개)
이 장을 마치면 학생들은:
1. AI 코딩 도구의 **특성과 한계**를 이해하고 설명할 수 있다
2. AI 도구의 **버전 불일치 문제**를 인식하고 해결 전략을 적용할 수 있다
3. **버전 명시 프롬프팅**과 **Context7 MCP**를 활용할 수 있다
4. AI 출력을 **검증하는 프로세스**를 수행할 수 있다
5. **GitHub Copilot**을 효과적으로 설정하고 활용할 수 있다

## 핵심 용어 (Key Terms)
- **LLM(Large Language Model)**: 대규모 언어 모델, AI 코딩 도구의 기반 기술
- **프롬프트(Prompt)**: AI에게 보내는 질문/요청
- **컨텍스트(Context)**: AI가 참조하는 문맥 정보
- **MCP(Model Context Protocol)**: AI에 외부 데이터를 연결하는 프로토콜
- **할루시네이션(Hallucination)**: AI가 사실과 다른 정보를 생성하는 현상

## 2.1 AI 코딩 도구의 특성과 한계 (5-6쪽)

### 집필 방향
AI 코딩 도구가 무엇이고, 어떤 원리로 동작하는지 학부생 눈높이에서 설명한다. 장점과 함께 반드시 알아야 할 한계를 균형있게 다룬다.

### 학습 순서 설계
1. **왜 배우는가**: "AI 도구를 잘 쓰면 생산성이 높아지지만, 잘못 쓰면 오히려 시간 낭비"
2. **무엇인가**: LLM 기반 코딩 도구의 동작 원리
3. **어떻게 사용하는가**: 효과적 활용을 위한 마인드셋

### 비유/예시 계획
- LLM → "책을 많이 읽은 조수: 지식은 많지만 최신 정보는 모름"
- 할루시네이션 → "자신있게 틀린 답을 하는 학생"

### 내용 구성

**2.1.1 AI 코딩 도구란?** (2쪽)
- 핵심 개념: ChatGPT, Copilot, Claude 등 주요 도구 소개
- LLM의 기본 원리 (학습 데이터 → 패턴 학습)
- 학생 질문 예상: "AI가 정말 코드를 이해하나요?"

**2.1.2 AI 코딩 도구의 한계** (3쪽)
- 핵심 개념: 학습 데이터 cutoff, 할루시네이션, 컨텍스트 제한
- 실제 오류 사례 (deprecated API 제안 등)
- 학생 질문 예상: "AI 코드를 그냥 복붙하면 안 되나요?"

### 참고문헌
- OpenAI. "GPT-4 Technical Report"
- GitHub. "Copilot Research"

## 2.2 버전 불일치 문제의 이해 (6-7쪽)

### 집필 방향
AI 도구를 사용할 때 가장 흔히 마주치는 문제인 버전 불일치를 구체적 사례와 함께 설명한다.

### 학습 순서 설계
1. **왜 배우는가**: "AI가 추천한 코드가 에러나는 이유"
2. **무엇인가**: 학습 시점과 현재 시점의 차이
3. **어떻게 사용하는가**: 문제 인식과 대처 방법

### 비유/예시 계획
- 버전 불일치 → "작년 지도를 보고 길 찾기: 새 건물이 안 나옴"
- 라이브러리 업데이트 → "앱 업데이트 후 메뉴 위치 변경"

### 내용 구성

**2.2.1 LLM 학습 시점과 라이브러리 버전 차이** (3쪽)
- 핵심 개념: 학습 데이터 cutoff date
- 예시: React 18, Supabase v2 등 버전 변화
- 표: 주요 AI 도구의 학습 cutoff 시점

**2.2.2 발생 가능한 오류 유형** (3쪽)

- 실제 오류 코드 예시

## 2.4 해결 전략 2: Context7 MCP 활용 (8-10쪽)

### 집필 방향
MCP(Model Context Protocol)의 개념을 소개하고, Context7을 활용하여 최신 문서를 AI에 연결하는 방법을 실습한다.

### 학습 순서 설계
1. **왜 배우는가**: "AI가 최신 공식 문서를 직접 참조하게 하면?"
2. **무엇인가**: MCP 개념과 Context7 도구
3. **어떻게 사용하는가**: 설치부터 활용까지 단계별 가이드

### 내용 구성

**2.4.1 MCP(Model Context Protocol) 개념** (3쪽)
- 핵심 개념: AI에 외부 데이터 소스 연결
- MCP의 동작 원리 (다이어그램)
- 비유: "AI에게 인터넷 검색 능력 부여"

**2.4.2 Context7 설치 및 설정** (3쪽)
- 핵심 개념: Claude Desktop + Context7 서버 연동
- 단계별 설치 가이드
- 트러블슈팅 팁

**2.4.3 최신 문서 참조 워크플로우** (3쪽)
- 핵심 개념: 문서 검색 → AI 질의 → 검증
- 실습: Supabase 최신 문서 참조하기
- 프롬프트 예시

## 2.5 AI 출력 검증 프로세스 (6-7쪽)

### 집필 방향
AI가 생성한 코드를 무조건 신뢰하지 않고, 체계적으로 검증하는 습관을 기르도록 한다.

### 학습 순서 설계
1. **왜 배우는가**: "AI 코드를 그대로 쓰면 생기는 문제들"
2. **무엇인가**: 검증 프로세스와 체크리스트
3. **어떻게 사용하는가**: 실무 검증 워크플로우

### 내용 구성

**2.5.1 공식 문서와 대조 검증** (2쪽)
- 핵심 개념: 공식 문서 = 진실의 원천
- 검증 우선순위: 공식 문서 > Stack Overflow > 블로그
- 실습: AI 코드와 공식 문서 대조

**2.5.2 AI 사용 로그 작성법** (2쪽)
- 핵심 개념: 프롬프트, 응답, 수정 사항 기록
- AI 사용 로그 템플릿
- 왜 기록해야 하는가 (학습, 디버깅, 평가)

**2.5.3 검증 체크리스트 템플릿** (2쪽)
- 핵심 개념: 체계적 검증 습관화
- 체크리스트 제공 (import 경로, API 버전, deprecated 여부 등)
