# 부록 E. 체크리스트 모음

이 부록에서는 웹 개발 프로젝트에서 자주 사용하는 체크리스트를 제공한다.

---

## E.1 접근성 체크리스트

### 기본 접근성

- [ ] 모든 이미지에 `alt` 속성이 있는가?
- [ ] 폼 요소에 `label`이 연결되어 있는가?
- [ ] 링크 텍스트가 의미 있는가? ("여기 클릭" 금지)
- [ ] 색상만으로 정보를 전달하지 않는가?
- [ ] 충분한 색상 대비가 있는가? (4.5:1 이상)

### 키보드 접근성

- [ ] Tab 키로 모든 인터랙티브 요소에 접근 가능한가?
- [ ] 포커스 표시가 명확한가?
- [ ] 키보드 함정(trap)이 없는가?
- [ ] Enter/Space로 버튼이 동작하는가?
- [ ] Esc로 모달/팝업을 닫을 수 있는가?

### 시맨틱 HTML

- [ ] 적절한 heading 계층 구조 (h1 → h2 → h3)
- [ ] 목록에 `ul`/`ol`/`li` 사용
- [ ] 표에 `table`, `th`, `caption` 사용
- [ ] 섹션에 `header`, `main`, `nav`, `footer` 사용
- [ ] 버튼에 `button` 태그 사용 (div 금지)

### ARIA 속성

- [ ] 동적 콘텐츠에 `aria-live` 적용
- [ ] 아이콘 버튼에 `aria-label` 적용
- [ ] 확장/축소에 `aria-expanded` 적용
- [ ] 로딩 상태에 `aria-busy` 적용

### 테스트

- [ ] 스크린 리더로 테스트 (NVDA, VoiceOver)
- [ ] 키보드만으로 전체 기능 사용 가능
- [ ] 200% 확대에서 레이아웃 유지
- [ ] Lighthouse 접근성 점수 90점 이상
