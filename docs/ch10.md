## 10.1 PostgreSQL 기초

### 10.1.1 관계형 데이터베이스 개념

**관계형 데이터베이스**(RDB, Relational Database)는 데이터를 **테이블** 형태로 저장하는 데이터베이스다. 엑셀 스프레드시트와 비슷하다고 생각하면 이해하기 쉽다.

Supabase는 **PostgreSQL**을 사용한다. PostgreSQL은 1996년부터 개발된 오픈소스 관계형 데이터베이스로, 안정성과 확장성이 뛰어나 대기업부터 스타트업까지 널리 사용된다.

**왜 관계형 데이터베이스인가?**

| 특징 | 설명 |
|------|------|
| 구조화된 데이터 | 테이블 스키마로 데이터 형식 강제 |
| 데이터 무결성 | 제약조건으로 잘못된 데이터 방지 |
| 관계 표현 | 테이블 간 연결로 복잡한 데이터 표현 |
| SQL 표준 | 표준화된 질의 언어 사용 |

### 10.1.2 테이블, 행, 열의 이해

```
┌─────────────────────────────────────────────────────────────┐
│                   관계형 데이터베이스 구조                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   테이블 (Table): posts                                     │
│   ┌────────┬─────────────┬─────────────┬──────────────┐    │
│   │   id   │    title    │   content   │  created_at  │◀─ 열(Column)
│   ├────────┼─────────────┼─────────────┼──────────────┤    │
│   │   1    │ 첫 번째 글  │ 안녕하세요  │ 2025-01-01   │◀─ 행(Row)
│   ├────────┼─────────────┼─────────────┼──────────────┤    │
│   │   2    │ 두 번째 글  │ 반갑습니다  │ 2025-01-02   │    │
│   └────────┴─────────────┴─────────────┴──────────────┘    │
│       ▲                                                     │
│       └─ Primary Key (기본 키)                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘

※ 그림 10.1 관계형 데이터베이스 구조
```

**핵심 용어**:

- **테이블(Table)**: 데이터를 저장하는 2차원 구조. 하나의 주제에 대한 데이터 집합
- **행(Row)**: 하나의 레코드. 게시글 하나, 사용자 한 명에 해당
- **열(Column)**: 데이터의 속성. 제목, 내용, 작성일 등
- **기본 키(Primary Key)**: 각 행을 고유하게 식별하는 열. 보통 `id`

### 10.1.3 기본 SQL 문법

Supabase 대시보드의 SQL Editor에서 직접 SQL을 실행할 수 있다. 기본 명령어를 알아두면 디버깅에 유용하다.

**조회 (SELECT)**:
```sql
-- 모든 게시글 조회
SELECT * FROM posts;

-- 특정 열만 조회
SELECT title, content FROM posts;

-- 조건 조회
SELECT * FROM posts WHERE id = 1;
```

**삽입 (INSERT)**:
```sql
INSERT INTO posts (title, content, user_id)
VALUES ('제목', '내용', 'uuid-here');
```

**수정 (UPDATE)**:
```sql
UPDATE posts
SET title = '새 제목'
WHERE id = 1;
```

**삭제 (DELETE)**:
```sql
DELETE FROM posts WHERE id = 1;
```

> **팁**: Supabase JavaScript 클라이언트는 이 SQL을 자동으로 생성해준다. SQL을 직접 작성할 일은 많지 않지만, 원리를 이해하면 디버깅이 쉬워진다.

## 10.2 데이터 모델링

### 10.2.1 게시판 스키마 설계

게시판을 만들려면 최소 세 개의 테이블이 필요하다:

1. **profiles**: 사용자 정보 (Supabase Auth와 연동)
2. **posts**: 게시글
3. **comments**: 댓글

```
┌─────────────────────────────────────────────────────────────┐
│                      게시판 ERD                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌───────────────┐         ┌───────────────┐              │
│   │   PROFILES    │         │    POSTS      │              │
│   ├───────────────┤         ├───────────────┤              │
│   │ id (PK)       │────┐    │ id (PK)       │              │
│   │ username      │    │    │ title         │              │
│   │ avatar_url    │    │    │ content       │              │
│   └───────────────┘    │    │ user_id (FK)  │◀─────┐       │
│         │              │    │ created_at    │      │       │
│         │              │    │ updated_at    │      │       │
│         │              │    └───────────────┘      │       │
│         │              │           │               │       │
│         │              └───────────┼───────────────┘       │
│         │                          │                       │
│         │              ┌───────────┘                       │
│         │              ▼                                   │
│         │    ┌───────────────┐                             │
│         │    │   COMMENTS    │                             │
│         │    ├───────────────┤                             │
│         │    │ id (PK)       │                             │
│         └────│ user_id (FK)  │                             │
│              │ post_id (FK)  │                             │
│              │ content       │                             │
│              │ created_at    │                             │
│              └───────────────┘                             │
│                                                             │
│   PK = Primary Key (기본 키)                                │
│   FK = Foreign Key (외래 키)                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘

※ 그림 10.2 게시판 ERD
```

### 10.2.2 테이블 관계: 1:N, N:M

**1:N 관계 (일대다)**:
- 한 사용자가 여러 게시글 작성 가능
- 한 게시글에 여러 댓글 작성 가능
- 가장 흔한 관계 유형

**N:M 관계 (다대다)**:
- 한 게시글에 여러 태그 가능
- 한 태그가 여러 게시글에 사용 가능
- 중간 테이블(연결 테이블)이 필요

**표 10.1** 관계 유형 비교

| 관계 | 예시 | 구현 방법 |
|------|------|----------|
| 1:1 | 사용자 - 프로필 | FK + UNIQUE |
| 1:N | 사용자 - 게시글 | FK만 사용 |
| N:M | 게시글 - 태그 | 중간 테이블 |

### 10.2.3 외래 키와 참조 무결성

**외래 키(Foreign Key)**는 다른 테이블의 기본 키를 참조하는 열이다.

`posts` 테이블의 `user_id`는 `profiles` 테이블의 `id`를 참조한다:

```sql
-- posts 테이블 생성
CREATE TABLE posts (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    title TEXT NOT NULL,
    content TEXT,
    user_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
    created_at TIMESTAMPTZ DEFAULT now(),
    updated_at TIMESTAMPTZ DEFAULT now()
);
```

**ON DELETE CASCADE**: 사용자가 삭제되면 해당 사용자의 게시글도 자동 삭제

**참조 무결성**: 존재하지 않는 사용자 ID로 게시글을 생성하려 하면 에러 발생. 데이터의 일관성을 보장한다.

### 10.2.4 Supabase에서 테이블 생성

**방법 1: 대시보드 UI**
1. Table Editor > New Table
2. 테이블 이름, 열 정의
3. 관계 설정

**방법 2: SQL Editor**
```sql
-- profiles 테이블 (Supabase Auth와 연동)
CREATE TABLE profiles (
    id UUID REFERENCES auth.users ON DELETE CASCADE PRIMARY KEY,
    username TEXT UNIQUE,
    avatar_url TEXT
);

-- posts 테이블
CREATE TABLE posts (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    title TEXT NOT NULL,
    content TEXT,
    user_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
    created_at TIMESTAMPTZ DEFAULT now(),
    updated_at TIMESTAMPTZ DEFAULT now()
);

-- comments 테이블
CREATE TABLE comments (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    content TEXT NOT NULL,
    post_id UUID REFERENCES posts(id) ON DELETE CASCADE,
    user_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
    created_at TIMESTAMPTZ DEFAULT now()
);
```

## 10.3 CRUD 구현

### 10.3.1 Supabase 쿼리 빌더

Supabase JavaScript 클라이언트는 **체이닝** 방식으로 쿼리를 작성한다:

```
┌─────────────────────────────────────────────────────────────┐
│                  Supabase 쿼리 빌더 흐름                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   supabase ──▶ .from('posts') ──▶ .select('*')             │
│                                         │                   │
│                                         ▼                   │
│                        ┌────────────────────────────┐       │
│                        │      필터 메서드           │       │
│                        │  .eq()  .neq()  .gt()     │       │
│                        │  .lt()  .like()  .ilike() │       │
│                        └────────────────────────────┘       │
│                                         │                   │
│                                         ▼                   │
│                               .order() ──▶ .range()        │
│                                              │               │
│                                              ▼               │
│                                          [실행]             │
│                                                             │
└─────────────────────────────────────────────────────────────┘

※ 그림 10.3 Supabase 쿼리 빌더 흐름
```

### 10.3.2 select: 데이터 조회

**전체 조회**:
```javascript
const { data, error } = await supabase
    .from('posts')
    .select('*');
```

**특정 열만 조회**:
```javascript
const { data, error } = await supabase
    .from('posts')
    .select('id, title, created_at');
```

**단일 행 조회** (`.single()`):
```javascript
const { data, error } = await supabase
    .from('posts')
    .select('*')
    .eq('id', postId)
    .single();  // 결과가 배열이 아닌 객체
```

**관계 데이터 조회** (조인 대체):
```javascript
const { data, error } = await supabase
    .from('posts')
    .select(`
        *,
        profiles ( username, avatar_url )
    `);
```

> **핵심**: Supabase는 외래 키 관계를 자동으로 감지하여 `profiles ( ... )` 형태로 관계 데이터를 가져올 수 있다.

### 10.3.3 insert: 데이터 생성

**단일 행 삽입**:
```javascript
const { data, error } = await supabase
    .from('posts')
    .insert({
        title: '새 게시글',
        content: '내용입니다',
        user_id: userId
    })
    .select()   // 생성된 데이터 반환
    .single();  // 객체로 반환
```

**여러 행 삽입**:
```javascript
const { data, error } = await supabase
    .from('posts')
    .insert([
        { title: '글 1', content: '내용 1', user_id: userId },
        { title: '글 2', content: '내용 2', user_id: userId }
    ])
    .select();
```

### 10.3.4 update: 데이터 수정

```javascript
const { data, error } = await supabase
    .from('posts')
    .update({
        title: '수정된 제목',
        content: '수정된 내용',
        updated_at: new Date().toISOString()
    })
    .eq('id', postId)  // 조건 필수!
    .select()
    .single();
```

> **주의**: `.eq()` 없이 update를 실행하면 **모든 행이 수정**된다!

### 10.3.5 delete: 데이터 삭제

```javascript
const { error } = await supabase
    .from('posts')
    .delete()
    .eq('id', postId);  // 조건 필수!
```

**표 10.2** CRUD 메서드 요약

| 작업 | 메서드 | 주의사항 |
|------|--------|----------|
| 조회 | `.select()` | 열 지정 가능, 관계 데이터 포함 가능 |
| 생성 | `.insert()` | `.select()` 추가로 생성된 데이터 반환 |
| 수정 | `.update()` | 조건 없으면 전체 수정 위험 |
| 삭제 | `.delete()` | 조건 없으면 전체 삭제 위험 |

## 10.4 쿼리 심화

### 10.4.1 필터링

**비교 연산자**:

```javascript
// 같음
.eq('status', 'published')

// 같지 않음
.neq('status', 'draft')

// 크다 / 작다
.gt('views', 100)   // views > 100
.gte('views', 100)  // views >= 100
.lt('views', 100)   // views < 100
.lte('views', 100)  // views <= 100
```

**문자열 검색**:
```javascript
// LIKE (대소문자 구분)
.like('title', '%React%')

// ILIKE (대소문자 무시) - 추천
.ilike('title', '%react%')
```

**배열 포함**:
```javascript
// 여러 값 중 하나
.in('category', ['tech', 'news', 'life'])
```

**복합 조건**:
```javascript
// OR 조건
.or('title.ilike.%react%,content.ilike.%react%')
```

**실전 예제 - 검색 기능**:
```javascript
async function searchPosts(keyword) {
    const { data, error } = await supabase
        .from('posts')
        .select('*')
        .ilike('title', `%${keyword}%`)
        .order('created_at', { ascending: false });

    if (error) throw error;
    return data;
}
```

### 10.4.2 정렬과 페이지네이션

**정렬**:
```javascript
// 최신순
.order('created_at', { ascending: false })

// 다중 정렬
.order('category', { ascending: true })
.order('created_at', { ascending: false })
```

**offset 기반 페이지네이션**:
```javascript
async function getPostsPaginated(page = 1, pageSize = 10) {
    const from = (page - 1) * pageSize;
    const to = from + pageSize - 1;

    const { data, error, count } = await supabase
        .from('posts')
        .select('*', { count: 'exact' })  // 전체 개수 조회
        .order('created_at', { ascending: false })
        .range(from, to);

    if (error) throw error;

    return {
        data,
        totalCount: count,
        totalPages: Math.ceil(count / pageSize),
        currentPage: page
    };
}
```

**표 10.3** 페이지네이션 방식 비교

| 방식 | 장점 | 단점 | 사용 시점 |
|------|------|------|----------|
| offset 기반 | 구현 간단, 페이지 이동 자유 | 대용량에서 성능 저하 | 소~중규모 데이터 |
| cursor 기반 | 대용량에서도 빠름 | 임의 페이지 이동 어려움 | 무한 스크롤, 대용량 |

**cursor 기반 페이지네이션** (대용량용):
```javascript
async function getPostsWithCursor(lastId = null, limit = 10) {
    let query = supabase
        .from('posts')
        .select('*')
        .order('id', { ascending: true })
        .limit(limit);

    if (lastId) {
        query = query.gt('id', lastId);
    }

    const { data, error } = await query;
    if (error) throw error;

    return {
        data,
        nextCursor: data.length > 0 ? data[data.length - 1].id : null,
        hasMore: data.length === limit
    };
}
```

### 10.4.3 관계 데이터 조회

Supabase의 강력한 기능 중 하나는 **한 번의 쿼리로 관계 데이터를 가져오는 것**이다.

**게시글 + 작성자**:
```javascript
const { data, error } = await supabase
    .from('posts')
    .select(`
        id,
        title,
        content,
        created_at,
        profiles (
            username,
            avatar_url
        )
    `);
```

**게시글 + 작성자 + 댓글** (중첩 관계):
```javascript
const { data, error } = await supabase
    .from('posts')
    .select(`
        *,
        profiles ( username, avatar_url ),
        comments (
            id,
            content,
            created_at,
            profiles ( username )
        )
    `)
    .eq('id', postId)
    .single();
```

**결과 구조**:
```javascript
{
    id: 'uuid',
    title: '게시글 제목',
    content: '내용',
    profiles: {
        username: '작성자',
        avatar_url: 'https://...'
    },
    comments: [
        {
            id: 'uuid',
            content: '댓글 내용',
            profiles: { username: '댓글 작성자' }
        }
    ]
}
```

> **핵심**: SQL의 복잡한 JOIN 없이도 관계 데이터를 직관적으로 가져올 수 있다. Supabase가 외래 키를 기반으로 자동으로 처리한다.

### 핵심 코드

**PostList 컴포넌트**:

```jsx
import { useState, useEffect } from 'react';
import { supabase } from '../lib/supabase';

export function PostList() {
    const [posts, setPosts] = useState([]);
    const [loading, setLoading] = useState(true);

    useEffect(() => {
        fetchPosts();
    }, []);

    async function fetchPosts() {
        try {
            const { data, error } = await supabase
                .from('posts')
                .select(`
                    *,
                    profiles ( username )
                `)
                .order('created_at', { ascending: false });

            if (error) throw error;
            setPosts(data);
        } catch (err) {
            console.error(err);
        } finally {
            setLoading(false);
        }
    }

    if (loading) return <p>로딩 중...</p>;

                    <p>작성자: {post.profiles?.username}</p>
```

```jsx
import { useState } from 'react';
import { supabase } from '../lib/supabase';
import { useAuth } from '../contexts/AuthContext';

export function PostForm({ onSuccess }) {
    const [title, setTitle] = useState('');
    const [content, setContent] = useState('');
    const [loading, setLoading] = useState(false);
    const { user } = useAuth();

    async function handleSubmit(e) {
        e.preventDefault();
        if (!user) {
            alert('로그인이 필요합니다.');
            return;
        }

        try {
            setLoading(true);
            const { data, error } = await supabase
                .from('posts')
                .insert({ title, content, user_id: user.id })
                .select()
                .single();

            if (error) throw error;
            setTitle('');
            setContent('');
            onSuccess?.(data);
        } catch (err) {
            alert('작성 실패: ' + err.message);
        } finally {
            setLoading(false);
        }
    }

    return (
        <form onSubmit={handleSubmit}>
            <input
                value={title}
                onChange={e => setTitle(e.target.value)}
                placeholder="제목"
                required
            />
            <textarea
                value={content}
                onChange={e => setContent(e.target.value)}
                placeholder="내용"
                required
            />
            <button disabled={loading}>
                {loading ? '저장 중...' : '작성하기'}
            </button>
        </form>
    );
}
```

## 핵심 정리

1. **관계형 데이터베이스**는 데이터를 테이블(행과 열) 형태로 저장한다. Supabase는 PostgreSQL을 사용한다.

2. **Primary Key**는 각 행을 고유하게 식별하고, **Foreign Key**는 다른 테이블을 참조하여 관계를 만든다.

3. **1:N 관계**는 외래 키만으로 구현하고, **N:M 관계**는 중간 테이블이 필요하다.

4. Supabase CRUD:
   - `.select()`: 조회, 관계 데이터 포함 가능
   - `.insert()`: 생성, `.select()`로 결과 반환
   - `.update()`: 수정, 조건 필수
   - `.delete()`: 삭제, 조건 필수

5. **필터링**: `.eq()`, `.neq()`, `.gt()`, `.lt()`, `.ilike()` 등으로 조건 지정

6. **페이지네이션**: `.range(from, to)`로 offset 기반, `.gt('id', lastId)`로 cursor 기반 구현

7. **관계 데이터 조회**: `profiles ( username )` 형태로 외래 키 관계를 따라 데이터를 한 번에 가져올 수 있다.
